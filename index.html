sonos one音箱与华硕路由器的连接冲突解决

title: sonos one音箱与华硕路由器的连接冲突解决

categories: 家居
tags:

- 路由器

1. 背景

意外获得sonos one音箱一枚，刚好家里缺一个在客厅放音乐的设备，于是非常开心的设置这个音箱，没想到一开始就遇到了一个wifi连接上的大问题

2. 问题描述

sonos one的接口极其稀少，连reset都没有，然后就是这个音箱只能通过两种方式连接：

  1. 家里的局域网wifi
  2. 苹果airplay（只有最新的sonos设备才支持airplay，支持列表请查询sonos官网）

airplay经常会找不到，连接的稳定性也极差（至少我的sonos one就是这么个情况），所以就必须要设置wifi连接的方式。但是我家的wifi-华硕ac88u居然无法设置成功，无论怎么弄就是无法让音箱连接到路由器的2.4Hz频段

3. 设置过程及问题解决方案

设置sonos one需要下载sonos的app（所有sonos设备都是如此），然后按照app上的指示一步一步操作即可。按照我的理解，这个app工作流程是这样的：

  1. 注册，这个不用说
  2. 通过蓝牙找到附近的sonos设备
  3. 通过蓝牙给sonos设备设置wifi
  4. 通过wifi尝试连接设备
  5. 基本设置完毕，然后是个性化的一些功能（比如添加QQ音乐）

华硕路由器就是在第4步的时候拉跨了，因为我把路由器的界面打开后发现，第三步是成功了的，路由器连接显示已经找到了sonos one这个设备，但是第4步尝试通过wifi连接设备的时候，sonos one又断开了与路由器的连接（我的sonos one还有一个更致命的问题，就是我直接通过网线连接路由器也不行），于是就造成了sonos的app上就永远的卡在了这一步骤上

解决方案：华硕ac88u的设置界面

  1. 进入华硕路由器的设置界面
  2. 选择左侧无线网络
  3. 选择专业设置
  4. 选择2.4Hz频段
  5. 停用无线传输公平性
  6. 点击应用此界面的设置


然后你再通过sonos的app来设置，比如我设置sonos one就一切顺利了

sonos one的小技巧 - 重置音箱：

  1. 断电
  2. 按住音箱屁股上的圆圆的按钮，一直按着，然后接通电源（接通电源后我仍然按着按钮的）
  3. 然后sonos one的灯首先会橘色跟白色交替闪烁，然后变成绿色闪烁（看到绿色闪烁的时候我才松开按钮），这样就重置了

4. 什么是无线传输公平性

目前使用的 2.4G频段 WiFi，遵守 802.11n 标准，他允许客户端同时使用多个天线同时收发信号，以使带宽翻倍。不过虽然 802.11n能够最高支持到 600Mbps带宽，但同一时间接入点只能与一个客户端通信。这就意味着，当多个客户端接入同一个无线路由器时，他们需要依次排队获得服务。当大家的接入速度相当时，并不会产生什么问题。但在实际情况下，客户端与接入点之间建立的无线连接会收到各种干扰的影响。当整个无线局域网内出现了慢速接入设备时，问题就发生了。
Airtime Fairness (发送时间公平性)技术通过转移部分慢速设备的服务时间给快速设备，来提高整个无线局域网的传输速度，优化等待时间，提高可用性，但会使得慢速设备变得更慢.
比如A与B同时接入一个无线接入点，A的连接速度是 1Mbps，B的连接速度是 5Mbps。当A需要传输 10M 数据时，B 不得不等待10秒才能够开始传输。在 AirtimeFairness 开启后，所有的客户端能够获得相等的服务时间。在本例中，A的前5秒内能够传输 5Mbps ，而后5秒将被分配给B，能够传输最高 25Mbps数据。所以在开启 AirtimeFairness 之后，整个无线局域网的传输性能获得了大幅提升，高性能设备获得了更高的连接速度。
当你的路由器同时连接大量无线客户端时，开启 AirtimeFairness 能够更好的在设备之间分配带宽，避免浪费以及慢速设备拖慢整个网络。另外，如果你的部分设备（例如手机）时常远离路由器并且信号欠佳时，也应该启用 AirtimeFairness 以保证其他设备的网络质量。

与很多的搭建blog的人一样，我也是很早就想搭建一个自己的blog，但是由于种种原因一直都没有弄起来，对于我个人来说有以下几个原因：

  1. 我不喜欢广告，希望blog只有文章、评论
  2. blog能支持繁杂的格式。比如各种代码能保持其在IDE一样的格式，可以打各种公式，以及你们现在看到的我在写这几个原因的这样的效果
  3. 能按照自己的想法来安排blog的样式，个人喜欢简洁、实用的风格
  4. 平台要足够稳定，因为国内很多平台都不稳定，随时可能倒闭，然后我就需要转移所有内容，这是我最为担心的。比如之前就是各种免费云，那么大的容量，1T、2T甚至无限的，最后统统关闭了，对这种具有中国特色的市场行为非常的反感
  5. 尝试过其他的平台都是一团糟，最后还是寻找国外的平台了

  • 安装git
  • 安装node.js
